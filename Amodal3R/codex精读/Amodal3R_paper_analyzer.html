Amodal3R: 从遮挡 2D 图像重建完整 3D 物体

一句话总结：Amodal3R 把“遮挡补全”从 2D 像素修补问题改写成 3D 条件生成问题：它不先补出一张完整 2D 图再重建 3D，而是在 TRELLIS 的 3D latent denoising 流程中显式注入可见区域 mask 和遮挡区域 mask，让模型直接生成完整、合理、纹理一致的 3D asset。

1. 研究背景与动机

单图像 3D 重建在过去几年快速发展，TRELLIS、Real3D、GaussianAnything、LaRa 等方法已经能从一张或多张图像中恢复高质量 3D 表示。但这些方法通常隐含一个前提：目标物体在输入图像里是完整可见的。真实场景很少满足这个条件，桌椅、玩具、车辆、包、食物等物体经常被其他物体遮挡。

如果输入只看到物体的一部分，传统 image-to-3D 模型往往会把“看不见”误读成“物体不存在”。于是生成的 3D 结果可能缺胳膊少腿，或者在遮挡边界附近出现破碎几何。这个问题不是普通图像重建的小瑕疵，而是任务定义本身的缺口：模型需要做的是 amodal reconstruction，也就是恢复被遮挡但客观存在的完整对象。

已有直观方案是两阶段 pipeline：先用 2D amodal completion 方法把遮挡图补成完整图，再把补全后的图送入 image-to-3D 模型。这种方式容易实现，但有两个根本问题。第一，2D 补全依赖图像外观先验，缺少真实 3D 几何约束。第二，多视角输入时，每个视角独立补全会产生不一致内容，后续 3D 重建器被互相冲突的视图误导。

Amodal3R 的出发点很清楚：不要把 3D 问题拆成一个不稳定的 2D 补全问题，再交给 3D 重建器收拾残局。它选择直接在 3D 生成模型内部处理遮挡，让模型同时完成可见部分重建、不可见部分推理和多视角一致性维护。

这些条件不是简单拼接到图像 token 后丢给模型，而是被放进两个专门机制：mask-weighted cross-attention 负责让模型更相信可见目标区域，occlusion-aware attention 负责告诉模型哪些区域是“可能存在目标但被挡住”的区域。

3. 预备知识：TRELLIS 与 3D latent diffusion

Amodal3R 不是从零设计 3D 生成器，它建立在 TRELLIS 之上。TRELLIS 的核心是 Structured Latent，也就是把 3D 对象表示成一组稀疏体素位置及其局部 latent feature：

$$z=\{(z_i,p_i)\}_{i=1}^{L}$$

其中 $p_i$ 是活跃体素位置，$z_i$ 是该位置上的局部 latent 特征。人话解释：模型只在物体表面附近放置少量有效 token，而不是在完整 $N^3$ 体素网格上浪费计算。

TRELLIS 包含两个扩散阶段：第一阶段预测 active voxel centers，也就是物体大体结构；第二阶段恢复对应的 structured latents，也就是更细的几何和外观。扩散训练目标写成：

$$\min_{\theta}\mathbb{E}_{(\ell^{(0)},x),t,\epsilon}\left\|\upsilon_{\theta}(\ell^{(t)},x,t)-(\epsilon-\ell^{(0)})\right\|_2^2$$

这里 $\ell^{(0)}$ 是真实 latent，$\ell^{(t)}$ 是加噪后的 latent，$\epsilon$ 是高斯噪声，$\upsilon_{\theta}$ 学习从 noisy latent 中预测去噪方向。人话解释：它在 3D latent space 里一步步把噪声变成结构化 3D 物体。

4. 核心创新一：Mask-weighted Cross-Attention

论文 Figure 3 展示了 Amodal3R 对 TRELLIS transformer block 的改造。第一个关键修改是 mask-weighted cross-attention。普通 cross-attention 会计算 latent query 和图像 token key 之间的相似度：

$$q=W_q\ell,\quad [k,v]=W_{kv}c_{dino},\quad S=\frac{qk^\top}{\sqrt{D}}$$

这里 $\ell$ 是 3D latent token，$c_{dino}$ 是 DINOv2 图像特征，$S$ 是每个 3D token 对每个图像 patch 的注意力打分。普通模型的问题是：遮挡物、背景、可见物体都会参与注意力，模型可能从错误区域读取条件。

Amodal3R 用可见 mask 生成 patch 权重 $c_{vis,j}\in[0,1]$，表示第 $j$ 个图像 patch 中目标可见像素所占比例。然后把这个权重放进 attention softmax：

$$A_{ij}=\frac{c_{vis,j}\exp(S_{ij})}{\sum_{k=1}^{K}c_{vis,k}\exp(S_{ik})}$$

人话解释：如果一个 patch 目标完全不可见，$c_{vis,j}=0$，它对 attention 的贡献就是 0；如果一个 patch 目标清晰可见，它的图像特征会被更强地利用。这个设计不是硬裁图，而是平滑地调制注意力，保留了 transformer 结构的可训练性。

这个机制的价值在于“少改动地利用 mask”。如果把 mask token 直接拼接给 transformer，预训练模型可能忽略这些低信息量 token，或者需要激进 fine-tuning 才能适配。mask-weighted attention 把 mask 作为 attention 权重的一部分，更直接、更稳定。

5. 核心创新二：Occlusion-aware Attention Layer

只有 $M_{vis}$ 还不够。不可见区域有两种含义：一种是纯背景，目标本来就不在那里；另一种是遮挡物前景，目标可能在后面继续延伸。Amodal3R 用 $M_{occ}$ 明确区分这两种情况。

方法是在 mask-weighted cross-attention 后面再接一个 occlusion-aware cross-attention layer，它单独读取 occlusion mask token $c_{occ}$。论文中的设计意图是：第一层尽量从可见区域提取可靠外观和结构线索，第二层再根据遮挡位置推理应该补全的隐藏部分。

附录 A.1 说明，输入图像分辨率为 $512\times512$，为了适配 DINOv2 patch size $14\times14$ 会 resize 到 $518\times518$，形成 $37\times37$ patch，再加 CLS token 和 register token，最终 $c_{dino}\in\mathbb{R}^{1374\times1024}$。mask 也被切成同样粒度，得到 $c_{vis}\in\mathbb{R}^{1374\times1}$ 和 $c_{occ}\in\mathbb{R}^{1374\times1}$。

多头版本在附录中写为：

$$A^h=\mathrm{softmax}(S^h+\log c_{vis})$$

$$\mathrm{MHA}=[A^1v;A^2v;\ldots;A^Hv]$$

人话解释：每个 attention head 都受到同一份可见性先验约束，但不同 head 仍然可以学习不同的几何或纹理关系。

6. 训练与遮挡模拟

真实世界中带完整 3D ground truth、真实遮挡、多视角一致标注的数据很难获得，因此作者选择合成训练。训练数据来自 3D-FUTURE、ABO 和 HSSD，共 20,627 个 synthetic 3D assets。附录说明 Amodal3R 在 4 张 A100 40G 上训练 20K steps，batch size 为 16，大约一天完成。

随机 2D 遮挡通过线条、圆、椭圆、矩形等形状叠加生成。更重要的是多视角评估中的 3D-consistent masks：作者从 3D mesh 上随机选取三角面，再用 random-walk 扩展为连续区域，并用相同相机参数渲染 mask。这样不同视角下的遮挡是几何一致的，能更真实地测试多视角重建。

多视角输入时，Amodal3R 不需要已知相机位姿，而是在 denoising 的不同 step 使用不同参考图像。因为早期 denoising 更影响粗几何，作者按可见面积 $|M_{vis}|$ 对视角排序，让可见信息更多的图像先参与生成。

9. 讨论：这篇论文真正推进了什么

Amodal3R 的核心贡献不是“用了 SAM”或“用了 TRELLIS”，而是把遮挡先验变成 3D 生成器内部的条件控制。它没有把遮挡当作输入图像上的坏像素，而是把遮挡当作生成过程必须理解的结构信息。

这个设计也解释了为什么两阶段方法在多视角时容易退化。2D completion 是每张图各自猜，猜错之后，3D 模型看到的是互相矛盾的证据。Amodal3R 直接在 3D latent space 汇合多视角证据，遮挡区域通过 mask 参与推理，而不是先被某个 2D 模型武断填满。

应用层面，Amodal3R 对机器人感知、3D scene decomposition、AR/VR asset creation 都有意义。现实场景里的物体经常互相遮挡，如果 3D 重建系统只能处理完整物体，它就很难真正进入复杂场景。

10. 局限与风险

数据规模有限

作者在附录 C.6 中明确指出，由于计算资源限制，训练数据只有 20,627 个 synthetic 3D assets，且主要集中在家具类别。复杂或分布外物体可能产生不真实结构。

真实世界适配不足

模型完全用 synthetic data 训练，无法充分利用真实环境中的上下文线索。真实遮挡、光照、接触关系和类别长尾仍可能造成失败。

补全不可控

当前遮挡部分长什么样由模型采样决定。用户无法用文本、草图或交互约束指定补全风格，这限制了创作和工业使用。

依赖 mask 质量

论文假设可以获得可见区域和遮挡区域 mask。真实使用中，如果 SAM 提示或分割结果错误，注意力引导也会偏离。

12. 结论

Amodal3R 的研究价值在于，它把 amodal completion 从“先补图再重建”的串联工程，提升为“遮挡感知的 3D 条件生成”。这种重写任务边界的思路，比单纯替换一个更强的 2D completion 模型更有生命力。

如果只记住一个点：Amodal3R 不是在问“遮挡区域像什么”，而是在问“在这个 3D 物体上，哪些部分被挡住了，以及它们应该如何与可见部分几何和外观一致”。这就是它相对两阶段 pipeline 的根本优势。

遮挡不是图像上的缺口，而是 3D 生成过程里的条件。
